Inrichting

  • Git - Git Flow, Subtree modules

    Commit op development branch en later merge naar master branch. Release van master branch naar release branch.

    Geen losse modules in de boom, dus atomic commits, wel history loss bij release.

  • Maven - Losse module boom, gegroepeerde "meta dependencies"

    Elke module wordt los gereleased, geen release dependency tangle.

    Meta dependencies detailleren deployments.

  • Jenkins - Delivery pipeline

    Clean - Compile - Test - Assets - Quality - Deliver - Integration Test

    Scrum board met taken

Workflow - CI (Jenkins)

Clean: Schoonmaken workspace

Compile: Compileren (integeren) van changes

Test: Uitvoeren Unit Tests (JUnit)

Assets: Maken van extra Assets (JavaDoc, Sources verzamelen, etc.)

Quality: Static & Dynamic code analysis met Sonar

Deliver: Upload naar Nexus

Integration Test: Uitvoeren van (complexere) Integratie/Systeem Tests (JUnit)

Workflow - Rework

Gefaalde Compilatie: Mail vanuit Jenkins, Appelflappen of Worstebroodjes

Gefaalde Unit Tests: Mail vanuit Jenkins, taak heropenen

Gefaalde Integratie Tests: Mail vanuit Jenkins, nieuwe taak om te herzien

Gefaalde Quality Gates: Jira taken op basis van Sonar analyse
